# 手机访问指南

## 方案一：局域网快速访问（零成本）

### 1. 启动应用

双击运行 `启动.bat`，看到控制台显示 `Network URL: http://192.168.x.x:8502` 即表示成功。

### 2. 查看电脑 IP 地址

在电脑上打开命令提示符（Win+R → 输入 `cmd`），运行：

```
ipconfig
```

找到 `IPv4 地址`，例如 `192.168.1.105`。

### 3. 手机访问

确保手机和电脑连接到 **同一个 WiFi**，然后在手机浏览器中输入：

```
http://192.168.1.105:8502
```

（将 `192.168.1.105` 替换为你电脑的实际 IP）

### 4. 添加到手机桌面

- **iPhone (Safari)**：打开网页 → 点击底部分享按钮 → "添加到主屏幕"
- **Android (Chrome)**：打开网页 → 点击菜单（⋮）→ "添加到主屏幕"

这样就像使用原生 App 一样方便！

## 方案二：ngrok 内网穿透（远程访问）

如果你需要在 **任何网络环境下** 访问，请使用 ngrok 方案。

详细步骤请查看 `ngrok部署方案.md` 文件。

**核心步骤：**
1. 下载 ngrok 并注册
2. 配置 Authtoken
3. 启动 ngrok 隧道：`ngrok http 8502`
4. 手机访问生成的公网地址

## 方案三：云端数据库同步（数据安全）

即使使用 ngrok 或局域网，也可以连接 Supabase 云数据库，实现数据云端同步：

1. 访问 https://supabase.com，用邮箱注册
2. 创建新项目，项目名填 `note-app`
3. 进入 **Project Settings** → **Database** → **Connection string** → 选择 **URI** 格式，复制连接字符串
4. 打开 `启动.bat`，找到并修改 `set DATABASE_URL=` 这一行：
   ```bat
   set DATABASE_URL=postgresql://postgres:你的密码@db.xxxxxxx.supabase.co:5432/postgres
   ```

   > 如果密码含 `@`、`:`、`?`、`#` 等特殊字符，需 URL 编码：
   > - `@` → `%40`
   > - `:` → `%3A`
   > - `?` → `%3F`
   > - `#` → `%23`

5. 保存后重新启动 `启动.bat`
6. 应用会自动检测 `DATABASE_URL` 并切换到 Supabase；如果未设置则继续用本地 SQLite

这样数据存储在云端，电脑关机也不影响数据安全。

## 方案对比

| 特性    | 方案一 (局域网) | 方案二 (ngrok) | 方案三 (ngrok + Supabase) |
| ----- | ---------- | ------------- | ------------------------ |
| 成本    | 免费         | 免费            | 免费                       |
| 配置时间  | 5 分钟       | 10 分钟         | 20 分钟                    |
| 远程访问  | 不支持        | 支持            | 支持                       |
| 多设备同步 | 不支持        | 不支持（数据在本地） | 支持                       |
| 数据安全  | 仅在本地       | 仅在本地          | 云端加密存储                 |

## 推荐路线

1. **先试试方案一**：改一行代码，立刻体验手机访问
2. **如果觉得好用**：配置 ngrok（方案二），实现远程访问
3. **数据安全要求高**：加上 Supabase 云端数据库（方案三）
