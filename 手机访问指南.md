# 手机访问指南

## 方案 A：局域网快速访问（零成本）

### 1. 启动应用

双击运行 `启动.bat`，看到控制台显示 `Network URL: http://192.168.x.x:8502` 即表示成功。

### 2. 查看电脑 IP 地址

在电脑上打开命令提示符（Win+R → 输入 `cmd`），运行：

```
ipconfig
```

找到 `IPv4 地址`，例如 `192.168.1.105`。

### 3. 手机访问

确保手机和电脑连接到 **同一个 WiFi**，然后在手机浏览器中输入：

```
http://192.168.1.105:8502
```

（将 `192.168.1.105` 替换为你电脑的 actual IP）

### 4. 添加到手机桌面

- **iPhone (Safari)**：打开网页 → 点击底部分享按钮 → "添加到主屏幕"
- **Android (Chrome)**：打开网页 → 点击菜单（⋮）→ "添加到主屏幕"

这样就像使用原生 App 一样方便！

## 方案 B：云端部署（真正多终端同步）

如果你需要在 **任何网络环境下** 访问，并且实现多设备数据同步，请使用云端部署方案。

### 架构图

```
手机  ──┐
平板  ──┼──→  Streamlit Cloud (免费托管)  →  Supabase PostgreSQL (免费云数据库)
电脑  ──┘
```

### 实施步骤

#### 第一步：注册 Supabase（云数据库）

1. 访问 [https://supabase.com](https://supabase.com)，用 GitHub/Google 账号注册
2. 创建新项目，项目名填 `note-app`
3. 进入 Project Settings → Database → 复制 `Connection string`（URI 格式）
4. 记录密码（Project Settings → Database → Password）Wlmm1234?1234

#### 第二步：修改 database.py 支持 PostgreSQL

将 `database.py` 中的 SQLite 连接替换为 PostgreSQL：

```python
import os
import psycopg2
from psycopg2.extras import RealDictCursor

# 从环境变量读取数据库连接信息（本地开发回退到 SQLite）
DATABASE_URL = os.environ.get("DATABASE_URL")

if DATABASE_URL:
    # 生产环境：PostgreSQL
    def get_conn():
        conn = psycopg2.connect(DATABASE_URL, cursor_factory=RealDictCursor)
        return conn
else:
    # 本地开发：SQLite（保持原有逻辑）
    import sqlite3
    from pathlib import Path
    DB_PATH = Path(__file__).parent / "data" / "notes.db"
    def get_conn():
        DB_PATH.parent.mkdir(parents=True, exist_ok=True)
        conn = sqlite3.connect(str(DB_PATH), check_same_thread=False)
        conn.row_factory = sqlite3.Row
        conn.execute("PRAGMA journal_mode=WAL")
        return conn
```

同时需要在 `requirements.txt` 添加 `psycopg2-binary`：

```
streamlit>=1.35.0
pandas>=2.0.0
plotly>=5.18.0
psycopg2-binary>=2.9.0
```

#### 第三步：调整 SQL 方言差异

PostgreSQL 和 SQLite 有少量语法差异，需要修改 `database.py`：

| SQLite                | PostgreSQL                                |
| --------------------- | ----------------------------------------- |
| `AUTOINCREMENT`       | `SERIAL` 或 `GENERATED ALWAYS AS IDENTITY` |
| `CURRENT_TIMESTAMP`   | `NOW()` 或 `CURRENT_TIMESTAMP`             |
| `?` 占位符               | `%s` 占位符                                  |
| `INTEGER PRIMARY KEY` | `SERIAL PRIMARY KEY`                      |

修改 `init_db()` 函数：

```python
def init_db():
    conn = get_conn()
    cur = conn.cursor()

    if DATABASE_URL:
        # PostgreSQL 建表语句
        cur.execute("""
            CREATE TABLE IF NOT EXISTS events (
                id SERIAL PRIMARY KEY,
                title TEXT NOT NULL,
                date DATE NOT NULL,
                start_time TEXT,
                end_time TEXT,
                location TEXT,
                note TEXT,
                color TEXT DEFAULT '#3780EA',
                created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
            )
        """)
        # ... 其他表类似
    else:
        # SQLite 建表语句（保持原有代码）
        ...

    conn.commit()
    conn.close()
```

#### 第四步：部署到 Streamlit Cloud

1. 将代码推送到 GitHub 仓库
2. 访问 [https://streamlit.io/cloud](https://streamlit.io/cloud)，用 GitHub 登录
3. 点击 "New app" → 选择你的 GitHub 仓库 → 选择 `note_app/app.py` 作为主文件
4. 在 "Advanced settings" 中添加环境变量：
   - `DATABASE_URL` = 你的 Supabase Connection URI
5. 点击 Deploy，等待部署完成

#### 第五步：验证

部署完成后，Streamlit Cloud 会给你一个 URL，如：

```
https://note-app-xxxx.streamlit.app
```

在手机、平板、电脑上都可以访问这个 URL，数据完全同步！

## 方案对比

| 特性    | 方案 A (局域网) | 方案 B (云端)                              |
| ----- | ---------- | -------------------------------------- |
| 成本    | 免费         | 免费（Supabase 免费额度 + Streamlit Cloud 免费） |
| 配置时间  | 5 分钟       | 30 分钟                                  |
| 远程访问  | 不支持        | 支持                                     |
| 多设备同步 | 不支持        | 支持                                     |
| 数据安全  | 仅在本地       | 云端加密存储                                 |
| 离线使用  | 需要本地网络     | 可以（需配合 PWA 缓存）                         |
