# 局域网连接问题排查手册

## 症状
手机浏览器显示 `net::ERR_CONNECTION_TIMED_OUT`，表示：
- 手机能找到电脑的 IP（192.168.2.103）
- 但连接被 **Windows 防火墙** 阻止了

---

## 第一步：运行防火墙放行脚本（90% 概率解决）

1. 在文件夹中找到 `防火墙放行.bat`
2. **右键点击** → 选择【**以管理员身份运行**】
3. 看到 "成功！端口 8502 已放行。" 即可

### 手动放行方式（如果脚本失败）
以管理员身份打开 **CMD**，粘贴执行：
```cmd
netsh advfirewall firewall add rule name="Streamlit-8502" dir=in action=allow protocol=TCP localport=8502
```

---

## 第二步：验证是否解决

1. 重新启动 `启动.bat`（保持运行）
2. 手机浏览器再次访问 `http://192.168.2.103:8502`

如果还是不行，继续第三步。

---

## 第三步：检查手机是否在同一 WiFi

**关键点**：手机截图中，状态栏同时显示了 **WiFi 图标** 和 **5G 信号**。手机可能优先使用了移动数据。

### 强制手机走 WiFi：
- **Android**：设置 → WLAN → 高级设置 → 关闭【智能切换网络】或【WLAN 助理】
- **iPhone**：设置 → 蜂窝网络 → 拉到最下面 → 关闭【无线局域网助理】

或者：临时关闭手机蜂窝数据（5G），只用 WiFi 再试。

---

## 第四步：路由器/AP 隔离检查

如果你的路由器开启了 **AP 隔离**（访客网络模式），会阻止局域网设备互访。

**检查方法**：
- 在手机浏览器中访问 `http://192.168.2.1`（路由器管理页）
- 如果能打开，说明手机确实在同一局域网
- 如果打不开，可能连错了 WiFi

**路由器设置**：登录路由器后台（通常是 192.168.2.1），查找以下选项并关闭：
- AP 隔离 / 客户端隔离
- 访客网络
- 智能路由优化

---

## 第五步：测试电脑到电脑的连通性

在另一台设备（或同一台电脑）上测试：

### 电脑上测试：
打开浏览器访问 `http://192.168.2.103:8502`
- 如果电脑自己都打不开：防火墙问题
- 如果电脑能打开但手机不行：路由器隔离或手机网络问题

### 命令行测试：
打开 CMD，运行：
```cmd
telnet 192.168.2.103 8502
```
如果提示 "连接失败" 或超时，说明端口被阻止。如果光标闪烁或空白，说明端口已通。

> Windows 11 默认没有 telnet，可以用 PowerShell 代替：
> ```powershell
> Test-NetConnection -ComputerName 192.168.2.103 -Port 8502
> ```
> 看 `TcpTestSucceeded` 是否为 `True`。

---

## 快速诊断清单

| 检查项 | 方法 | 期望结果 |
|--------|------|----------|
| 本机访问 | `http://localhost:8502` | 能打开 |
| 本机IP访问 | `http://192.168.2.103:8502` | 能打开 |
| 防火墙规则 | `netsh advfirewall firewall show rule name="Streamlit-8502"` | 显示已启用 |
| 端口监听 | 控制台显示 `0.0.0.0:8502` | 确认已监听 |
| 手机Ping | 手机CMD/Ping App 测试 `192.168.2.103` | 能Ping通 |
| 手机端口 | 手机浏览器访问 `http://192.168.2.103:8502` | **问题所在** |

## 备用方案：如果局域网实在搞不定

直接部署 **云端方案**（真正解决所有问题）：

1. 注册 [Supabase](https://supabase.com)（免费，500MB 数据库）
2. 把 `database.py` 替换为 `database_pg.py`
3. 部署到 [Streamlit Cloud](https://streamlit.io/cloud)（免费）
4. 得到一个 `https://xxx.streamlit.app` 的 URL
5. 手机、电脑、平板，任何地方都能访问，数据完全同步

详见 `手机访问指南.md` 中的【方案 B：云端部署】。

## 常见错误对照

| 错误信息 | 原因 | 解决 |
|----------|------|------|
| `ERR_CONNECTION_TIMED_OUT` | 防火墙阻止 | 运行防火墙放行脚本 |
| `ERR_CONNECTION_REFUSED` | 应用没启动或端口错误 | 检查启动.bat是否运行 |
| `ERR_ADDRESS_UNREACHABLE` | 手机不在同一局域网 | 检查WiFi是否相同 |
| `DNS_PROBE_FINISHED_NXDOMAIN` | 地址写错了 | 检查IP地址是否正确 |
