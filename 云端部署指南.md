# 云端部署指南（手把手步骤）

部署完成后，你将得到一个 `https://xxx.streamlit.app` 的 URL，手机、电脑、平板在任何网络下都能访问，数据完全同步。

---

## 第一步：注册 GitHub 并推送代码（约 5 分钟）

### 1.1 注册 GitHub 账号
1. 访问 https://github.com/signup
2. 用邮箱注册，按提示完成验证
3. 记住你的用户名（如 `lin-note`）

### 1.2 安装 Git（如果电脑没有）
1. 访问 https://git-scm.com/download/win
2. 下载安装，全部默认选项下一步即可
3. 安装完成后，打开 CMD 验证：
   ```
   git --version
   ```
   看到版本号即表示安装成功

### 1.3 推送代码到 GitHub
在 `note_app` 文件夹内，右键选择 **Git Bash Here**（或打开 CMD 进入该目录）：

```bash
# 1. 初始化 Git 仓库
git init

# 2. 添加所有文件
git add .

# 3. 提交代码
git commit -m "first commit"

# 4. 连接远程仓库（把 USERNAME 换成你的 GitHub 用户名）
git remote add origin https://github.com/USERNAME/note-app.git

# 5. 推送到 GitHub
git branch -M main
git push -u origin main
```

> 第 4 步执行前，需要先在 GitHub 网站上创建一个名为 `note-app` 的空仓库：
> 1. 登录 GitHub → 点击右上角 `+` → `New repository`
> 2. 仓库名填 `note-app`
> 3. 选择 `Public`（免费）
> 4. 不要勾选 README 或 .gitignore（本地已准备好）
> 5. 点击 `Create repository`

## 第二步：注册 Supabase（云数据库，约 5 分钟）

### 2.1 注册账号
1. 访问 https://supabase.com
2. 点击 `Start your project`，用 GitHub 账号登录
3. 等待创建组织

### 2.2 创建项目
1. 点击 `New project`
2. 项目名称：`note-app`
3. 数据库密码：设置一个复杂密码并**记住**（后续要用）
4. 区域选择：Asia Pacific (Singapore) 或 Asia Pacific (Mumbai)
5. 点击 `Create new project`，等待约 1-2 分钟

### 2.3 获取数据库连接地址

**方法 A：直接复制 URI（推荐）**

1. 项目创建完成后，进入项目首页
2. 点击左侧 `Database`（数据库图标）
3. 点击上方 `Connect` 标签
4. 在 `Connection string` 区域，选择 **URI** 格式
5. 点击 `Copy` 复制，格式如下：
   ```
   postgresql://postgres:[YOUR-PASSWORD]@db.xxxxxxx.supabase.co:5432/postgres
   ```
6. 把 `[YOUR-PASSWORD]` 替换为你刚才设置的密码
7. **把这串地址保存好**，下一步要用

> **找不到 URI 选项？** 看方法 B：

**方法 B：手动组装连接字符串**

如果界面上只有 `Host`、`Port`、`Database` 等分散信息，按以下格式手动组装：

```
postgresql://postgres:你的密码@db.xxxxxxx.supabase.co:5432/postgres
```

其中 `xxxxxxx` 是你的项目 **Reference ID**，按以下方式找到：

1. 点击左侧 `Project Settings`（齿轮图标）→ `General`
2. 在 `Project ID` 或 `Reference ID` 处，找到类似 `abcdefghijklmno` 的字符串
3. 填入上面的 `xxxxxxx` 位置即可

**示例：**
如果你的密码是 `MySecret123`，Reference ID 是 `abc123def456`，则连接字符串为：
```
postgresql://postgres:MySecret123@db.abc123def456.supabase.co:5432/postgres
```

> 注意：这串地址包含密码，不要泄露给别人，也不要直接写在代码里。

## 第三步：部署到 Streamlit Cloud（约 5 分钟）

### 3.1 注册 Streamlit Cloud
1. 访问 https://streamlit.io/cloud
2. 点击 `Sign up with GitHub`，用同一个 GitHub 账号登录

### 3.2 部署应用
1. 点击 `New app`
2. 选择你刚才创建的 GitHub 仓库：`note-app`
3. Branch 选择 `main`
4. Main file path 填：`app.py`
5. 点击 `Advanced settings` 展开
6. 在 **Secrets** 区域添加环境变量：
   - 点击 `+ New secret`
   - Key 填：`DATABASE_URL`
   - Value 填：刚才保存的 Supabase 连接地址（`postgresql://...`）
   - 点击 `Save`
7. 点击 `Deploy!`

### 3.3 等待部署
- 部署过程约 1-3 分钟
- 看到绿色的 `Your app is in the oven!` 表示正在构建
- 完成后会自动跳转到你的应用页面，URL 类似：
  ```
  https://note-app-username.streamlit.app
  ```

## 第四步：验证（2 分钟）

## 常见问题

### Q1：部署后页面空白或报错？
检查 Streamlit Cloud 的日志：
1. 打开你的应用管理页
2. 点击右下角 `Manage app` → `Logs`
3. 查看报错信息，通常是：
   - `DATABASE_URL` 填错了 → 检查密码和地址
   - 依赖没安装 → 检查 `requirements.txt` 是否包含 `psycopg2-binary`

### Q2：如何更新应用？
本地修改代码后，重新推送：
```bash
git add .
git commit -m "update"
git push
```
Streamlit Cloud 会自动检测更新并重新部署。

### Q3：如何回退到本地使用？
把 `database_local.py` 复制回 `database.py` 即可：
```bash
cp database_local.py database.py
```

### Q4：数据安全吗？
- Supabase 数据库支持 SSL 加密传输
- 数据存储在云端，比本地硬盘更安全（不会丢失）
- 建议定期导出 JSON 备份（应用内有导出功能）
